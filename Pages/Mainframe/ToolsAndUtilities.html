Tools and Utilities

  • ENDEVOR: ENDEVOR is an integrated set of management tools used to automate,Control, and monitor your applications development process. ENDEVOR runs under MVS, within the TSO/ISPF environment, and in batch.
  • Control-M: Control-M is a workload automation software i.e. it is a batch scheduling software. We can schedule workloads on daily, weekly or monthly intervals depending on our requirements. Control-M can automate JCL, batch files, shell scripts, database stored procedures, file transfer and web services functions.
  • File Aid: This tool is used to browse, edit, allocate, compare, copy, delete, and print files of any standard MVS access method. For a partitioned database (PDS), you can browse, edit, and copy an individual member or the entire dataset. We can do all these activities on VSAM files also. It provides a formatted mode that allows you to browse and edit one record at a time, field-by-field, using your COBOL or PL/I record layout field names to describe each field. It also provides a vertical formatted mode that allows you to browse and edit records on a field-by-field basis using the record layout field names as column headers. It also allows us to filter the file data on different conditions (consisting of AND and OR conditions).
  • SORT: SORT is a mainframe utility which allows us to sort data in a file. It also helps us format data in a file. SORT utility can also filter data on basis of different conditions.
  • FTP: File Transfer Protocol is a mainframe tool which can be used for transferring mainframe files from one region to different region.
  • Xpeditor: Xpeditor is a mainframe debugging tool used to understand execution flow of a program. This can be used to debug both batch and online programs.
  • ICETOOL: ICETOOL is a mainframe utility that uses capabilities of DFSORT to perform multiple sort and formatting operations on one or more datasets in a single step.
  • IEBGENER: IEBGENER is a mainframe utility which allows us to copy one dataset to another. IEBGENER also can filter data; change a data set's logical record length (LRECL) and block size (BLKSIZE); and generate records.
