What is EAF4J?

  • The Enterprise Application Framework for Java (EAF4J) is a UHG-owned Java application framework to facilitate rapid, robust, high-quality software delivery by application development groups.
  • It provides extendable components implementing commonly used functionality for construction of POJO based enterprise applications and is aligned with UHG-specific design patterns and technology standards.
  • It is not a scaffolding framework for application auto generation from domain models (e.g. Groovy Grails)

EAF4J Provides

  • Recommended technology stack leveraging industry standards and leading open-source libraries
  • A set of Java libraries containing
    • functional components for common functions such as caching, persistence, transaction management, security, validation, and web presentation and flow
    • custom UHG extensions that codify common UHG design patterns and infrastructure standards
  • Sample applications illustrating framework use and serving as the basis for custom development
  • Other Knowledge Resources
    • Developer guides
    • Formalized training
    • UHG Java Community discussion forums

Extensibility

  • EAF4J provides default implementations meeting the requirements of many application teams
  • Being a framework, EAF4J is extendable, providing architects and developers flexibility to implement application specific requirements when needed

Business Benefits

  • Avoid time & cost of evaluating frameworks & tools for use in application development efforts
  • Simplify developer set-up time with pre-defined IDE configuration and plug-ins
  • Reduce project risk

    • Leveraging lessons learned by other groups in UHG
    • Building off of proven solutions instead of starting from scratch
  • Internal support of EAF4J framework
    • Free support via internal discussion forums

  • Modern technologies readily available in the marketplace

EAF4J Technology Stack

EAF4J Packaging

The functionality of EAF4J is packaged as a set of modular Java libraries that allow applications to select only those functional components they require and minimize the open source dependencies.

Core framework components

  • Provides entry point to Spring IOC container
  • Foundation of all other framework modules
  • Includes common application components
    • Logging, including components for enterprise logging pattern
    • Exception handling
    • Encryption in alignment with IRM standards
    • Spring context loading via JVM parameters

EAF4J Cache

Storage & access to frequently used data
  • Based upon OpenSymphony OS Cache
  • Static cache support
    • Frequently-accessed, yet infrequently modified data
  • Dynamic caching support via AOP
    • Frequently-accessed, frequently modified data
  • Support for clustered applications
  • Configurable cache loading
    • Pre-load cache(s) on startup
    • Lazy-load caches when first accessed

EAF4J Core

  • JVM parameter controlled Spring context loading
    • Allows environment specific context configuration files.
  • Core Exception Handling
    • Exception severity and handling based upon EAF4J BaseException
      • extends java.lang.RuntimeException.
    • BaseException links to error messages via Spring Resource Bundle.
    • Extensible ErrorHandler performs standard steps
      • Identify the type of exception and get the corresponding error code
      • Use Spring ResourceBundleMessageSource to get Error messages from properties file for the given error code.
      • Log the exception. The underlying logging mechanism is hidden and uses EAF4J logging framework for writing logs.
  • Enterprise Logging Pattern Implementation

Enterprise Logging Pattern Implementation

Log using a standard UHG XML Layout
  • com.uhg.eaf4j.core.logging.layout. UHGXMLLayout
  • Extends log4j XMLLayout adding Application ID and Logtype to header elements.
Universal Transaction ID Generator
  • com.uhg.eaf4j.core.logging.uid.UniversalTransactionIDGen
  • Generates a universal id that can be used for logging as a transaction passes between systems, so that each system uses the same id within their logging.

EAF4J Java API Example

EAF4J XML Layout Configuration
<layout class="com.uhg.eaf4j.core.logging.layout.UHGXMLLayout">
<param name="Properties" value="TRUE" />
<param name="ApplicationId" value="ApplicationsID" />
<param name="defaultLogType" value="genericLogMessage" />
</layout>
Universal Transaction ID Generation
<bean id="universalIdGen" class="com.uhg.eaf4j.core.logging.uid.UniversalTransactionIDGen">
<property name="applicationId" value=“ApplicationsID" />
</bean>

//Inject Universal Transaction ID bean into idGenor get directly
UniversalTransactionIDGen idGen;
String transactionType = “Claim";
String transactionUid = idGen.getTransactionId(transactionType);
